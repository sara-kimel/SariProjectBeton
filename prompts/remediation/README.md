# פרויקט שרי — פרומפטים לתיקונים (Remediation Track)

תיקייה זו מכילה **פרומפט עצמאי לכל שלב תיקון** שנגזר מ**סקירת הקוד המקיפה** על ה-MVP הבנוי (ראה `SPEC.md` §18). בדומה ל-`prompts/` המקורי, כל קובץ נבנה כדי להיפתח **בצ'אט חדש** ומכיל את כל ההקשר, משימות חוצות DB→שרת→לקוח, ובדיקות מקצה-לקצה.

> **ההבדל מ-`prompts/phase-0..6`:** אותם שלבים *בנו* את ה-MVP. השלבים כאן (R0..R8) *מתקנים ומקשיחים* אותו לאחר סקירה. מסמך האמת: `SPEC.md` §18 (טבלת P0 + הקשחות P1 + השלמות P2), עם גרסה קריאה ב-`SPEC.html`.

## איך משתמשים
1. פותחים צ'אט חדש של Claude Code **בתיקיית הפרויקט**.
2. כותבים: `בצע את השלב לפי prompts/remediation/R?-....md`.
3. הסוכן יקרא את `SPEC.md` (§18 + הסעיפים הרלוונטיים), את `prompts/README.md` (הקשר מאסטר) ואת קובץ ה-README הזה, ואז יבצע.
4. **סדר:** R0 כבר בוצע. המשך לפי עדיפות — R1→R7 (P1 ואז P2); R8 אופציונלי/עתידי. השלבים ברובם עצמאיים; אין תלות קשיחה ביניהם מלבד ש-R0 קדם לכולם.
5. בסוף כל שלב: הרצת חבילת הבדיקות (`pytest`) + `npm run build`, סימון הפריטים ✅ ב-`SPEC.md` §18 (ובמראה `SPEC.html`), ועדכון הזיכרון.

## הקשר מאסטר
כל ההקשר המלא (מה הפרויקט, ההחלטות שננעלו OD-1..OD-13, המחסנית, מפת הקבצים, מוסכמות עבודה, גישת בדיקות) נמצא ב-**`prompts/README.md`** — לקרוא אותו קודם. תמצית מצב:
- **ה-MVP המלא (phases 0-6) בנוי ועובד.** חבילת בדיקות שרת = **52 עוברות** (`cd server && PYTHONUTF8=1 ./.venv/Scripts/python.exe -m pytest -q`; דורש SQL Server מקומי + DB `beton`).
- **✅ כל 6 תיקוני ה-P0 בוצעו ואומתו** — ראה `R0-critical-fixes-DONE.md`.
- **שינוי סמנטי חשוב מ-R0:** `DELETE` על בקשה/פנייה כעת = **ביטול רך** (`status='CANCELLED'`, הרשומה נשמרת, GET מחזיר 200) — לא מחיקה פיזית.

## הגדרת סיום (בכל שלב)
1. כל משימות ה-DB/שרת/לקוח בוצעו לפי הקובץ.
2. **חבילת ה-pytest ירוקה** (בלי regressions), וגם `npm run build` בלקוח.
3. עדכון `SPEC.md` §18 — סימון הפריטים שבוצעו ✅ (ובמראה `SPEC.html`).
4. עדכון הזיכרון (`memory/project-sari-state.md`).
5. סיכום קצר למשתמש: מה תוקן, איך נבדק, ומה הבא.
